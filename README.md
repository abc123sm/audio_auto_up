# Audio Auto Up (全自动同人音声搬运/发布工作流)

这是一个专为同人音声（RJ/VJ资源）设计的**半自动化抓取、下载、解压、做种、发布与辅种**工具集。本工具旨在极大减少从 南+ 获取资源并搬运发布至 KamePT 的人工干预成本。

⭐ **如果这个项目极大地节省了您的时间，请不要吝啬您的 Star，您的支持是我持续更新与开源的动力！** ⭐

## 🌟 核心特性

- **智能论坛爬虫**：自动解析南+等论坛帖子，智能提取百度网盘/MEGA下载链接。
- **自动购买判定**：自动识别帖子售价，对符合心理预期价格（如免费或极低售价）的资源自动执行购买逻辑，默认购买6sp以下资源。
- **智能密码抓取与解压**：基于上下文和正则表达式，精准提取针对当前资源的解压密码候选列表，下载完成后全自动调用 7z 进行精准试错解压。目前已集成部分常见的视频隐写（如伪装成 MP4/MKV 的压缩包）自动解密功能。
- **全链路自动发布**：自动生成 `.torrent` 种子文件，并通过 **kamegen** 自动获取简介，一键上传发布到 KamePT。
- **自动挂机辅种**：发布完成后，自动调用本地 qBittorrent 的 WebUI 接口，将新种子无缝推送到下载器中开始做种。

## ⚙️ 工作流程

本工具由多个模块解耦组成，推荐的运行流程如下：

1. **资源抓取 (`crawler.py`)**：抓取论坛页面，自动过滤高昂售价（≥6 SP）帖子并自动完成购买。提取 `RJ号`、百度/MEGA链接、提取码以及潜在的解压密码（记录到 `jieya.txt` 中）。
   - 默认从第 1 页开始抓取 10 页数据。
   - **运行指令**：
     - `python crawler.py` （默认运行）
     - `python crawler.py <起始页码> <抓取总页数>` （例如：`python crawler.py 1 20`）
2. **人工核对数据（重要）**：抓取完成后，请打开 `dl_list.txt`（百度网盘）和 `mega.txt`（MEGA网盘），人工核对是否存在抓取错误。
   - 需要留意排查的常见错误：抓取到了自购证明图片链接、提取码抓错、解压密码抓错等。及时修正可避免后续下载或解压报错。
3. **网盘下载 (`pcsd.py` / `mega_manager.py`)**：调用下载模块，模块会自动创建以 RJ 号命名的文件夹并下载资源。下载完成后会读取抓取阶段生成的专属密码列表，自动调用 7z 进行精准试错解压。
   - 如果硬盘空间不足，`pcsd.py` 支持指定单次处理的下载数量，方便分批次下载与发布。
   - **运行指令**：
     - `python pcsd.py` （默认仅下载 `dl_list.txt` 的前 5 个资源）
     - `python pcsd.py <下载数量>` （例如：`python pcsd.py 10`）
     - `python mega_manager.py` （自动读取 `mega.txt` 队列下载 MEGA 资源，内部已集成针对伪装格式如 pdf/png 的大文件自动重命名解压逻辑）
4. **人工检视（重要）**：确认解压后的文件及目录结构是否正常。
   - **手动解压**：如果遇到解压失败的文件（例如复杂的反和谐加密、多重分卷解压等工具尚不支持的情况），请手动进行解压。程序目前仅支持部分常见的隐写解密、修改副档名等，如果遇到无法处理的情况，可参考 `dl_list.txt` 或 `mega.txt` 中记录的论坛原帖链接（即第 5 列），查看对应文章进行手动处理。
   - **清理并整理目录**：解压出的文件夹如果层级混乱，或者包含不需要的广告、快捷方式等无关文件，请在制作种子前手动整理并删除。
   - **修正编码**：如果解压出的文件名有乱码，请使用 [cntojp](https://github.com/CepKana/CNtoJP) 工具进行修正。该工具可以处理大部分的jis->gbk编码问题。
5. **转移文件至待发布资料夹**：将清理好的资料夹转移至 `torrent/` 目录下。
6. **一键发布做种 (`main_workflow.py`)**：执行最终的做种与发布自动化流水线。
   - **运行指令**：`python main_workflow.py`
   - 发布器会自动执行以下核心流程：
     1. 深度清理 `torrent/` 文件夹中由系统或 Mac 产生的垃圾残留（如 `Thumbs.db`, `desktop.ini`, `.DS_Store`, `.AppleDouble`, `.localized` 等）。
     2. 针对 `torrent/` 目录下的每个资源文件夹制作 `.torrent` 种子文件。
     3. 制作完毕后，将源文件夹转移到 `file/` 目录中，防止下次重复制作和发布。
     4. 自动上传新种子文件，并抓取 KamePT 等平台相关简介完成发布。
     5. 自动将发布成功的种子推送到本地 qBittorrent 客户端开始下载辅种。

## 🛠️ 环境依赖与配置

1. **Python 环境**：推荐 Python 3.8+，并安装 `requests`, `beautifulsoup4` 等基础依赖。
2. **外部组件**：
   - **BaiduPCS-Go**：本项目依赖 [BaiduPCS-Go](https://github.com/qjfoidnh/BaiduPCS-Go) 进行百度网盘下载，请将其 `BaiduPCS-Go.exe` 放置在项目目录或配置正确的路径。
   - **MegaTools**：本项目依赖 [MegaTools](https://megatools.megous.com/) 进行 MEGA 网盘下载，请下载并将其添加到系统环境变量中。
   - **7-Zip**：用于自动解压，需在系统中安装并配置好路径。 推荐使用这个7z版本 [7z-zstd](https://github.com/mcmilk/7-Zip-zstd)
   - **qBittorrent**：需要开启 WebUI 功能，用于接收并辅种最后生成的种子。
   - **cntojp**：用于修正解压出的文件名乱码，请下载并将其添加到系统环境变量中。
   - **MKVToolNix (可选)**：如果需要支持极端罕见的 MKV 附件格式隐写解密，请在项目根目录新建 `tools` 文件夹，并放入 `mkvextract.exe` 和 `mkvinfo.exe`。普通的 MP4 隐写解密为纯 Python 实现，无需此工具。

### 初始配置指南

1. 将项目根目录下的 `config.example.py` 复制一份并重命名为 `config.py`。
2. 使用编辑器打开 `config.py`，根据注释填写您的个人信息：
   - `BASE_DIR`：项目绝对路径。
   - `SEVEN_ZIP_PATH` / `PCS_PATH`：7z和百度网盘下载器的路径。
   - `BAIDU_USERNAME` / `BAIDU_DOWNLOAD_PATH`：百度网盘账号与文件保存路径（存到百度盘的）。
   - `QB_URL` / `QB_USER` / `QB_PASS`：qBittorrent 的 WebUI 地址与账密。
3. 准备账号凭证文件：
   - `nanjia.txt`：用于存放 南+ 的 Cookies。
   - `kamept_cookies.txt`：用于存放 KamePT 的 Cookies。
4. 登入BaiduPCS-Go

## 🍪 Cookies 获取教程

爬虫模块与自动上传模块都需要您提供对应网站的登录凭证 (Cookies)。标准的获取流程如下：

1. 在浏览器（Chrome / Edge）中打开并**登录**对应的目标网站（如 南+ 或 KamePT）。
2. 按下键盘上的 `F12` 键打开“开发者工具”。
3. 切换到顶部的 **Network（网络）** 标签页。
4. 在页面内随便点击一个链接，或者直接按 `F5` 刷新页面，此时网络面板会抓取到一堆请求。
5. 在左侧的请求列表中，找到主要的 HTML 文档请求（通常是最顶部的第一个请求，或者名称与当前 URL 相同的请求）。
6. 右键点击该请求，选择 **Copy (复制)** > **Copy as cURL (posix) (复制为 cURL (posix))**。
   - 浏览器不同可能显示的文字也不同，也可以是 **Copy as cURL (bash)**
7. 将获取到的这长串代码，直接粘贴并保存到项目根目录对应的文本文件中（如 `nanjia.txt` 或 `kamept_cookies.txt`）。

## 目前项目的主要缺陷，并且想不到如何优化
1. 耦合度不是很高，很多环节需要人工介入，但反和谐手段太多了，然后资源发布者也不一定一篇文章就对应一个rj号
